动态规划问题

2021-01-17
LeetCode
  • Algorithms

斐波那契数列

递归写法

int fib(int N) {
   if (N == 1 || N == 2) return 1;
   return fib(N - 1) + fib(N - 2);
}

递归算法的时间复杂度：子问题个数乘以解决一个子问题需要的时间
子问题个数： O(2^n)
解决一个子问题的时间：在本算法中没有循环，只有f(n-1)+f(n-2)这一个加法操作，所以时间复杂度为： O(1)
所以该算法的时间复杂度为： O(2^n)
通过递归树可以发现有很多的重复计算，如：f(18)。

这就是动态规划问题的第一个性质：重叠子问题。

带备忘录的递归解法

造一个备忘录，每遇到一个子问题，先去“备忘录”里查看。

int fib(int N) {
    if (N < 1) return 0;
    vector<int> memo(N + 1, 0);
    return helper(memo, N);
    }

int helper(vector<int> & memo, int n) {
    if (n == 2 || n == 1) return 1;
    if (memo[n] != 0) return memo[n];
    memo[n] = helper(memo, n - 1) + helper(memo, n - 2);
    return memo[n];
}

通过递归树可以发现，已经极大地减少了节点个数。
子问题个数：图中节点的总数：O(n)
解决一个子问题的时间：O(1)
时间复杂度：O(n)
这种解法叫做 “自顶向下”，动态规划叫做：“自底向上”。

自顶向下：画递归树的时候，是从上向下延伸的。
动态规划：从问题规模最小的 f(1) 和 f(2) 开始向上推，直到最后的 f(20)，这就是动态规划的思路。
动态规划一般都脱离了递归，而是由循环迭代完成。

DP数组的迭代解法

int fib(int N) {
    vector<int> DP(N + 1, 0);
    DP[1] = DP[2] = 1;
    for (int i = 3; i <= N; i++) {
        DP[i] = DP[i - 1] + DP[i - 2];
        return DP[N];
    }
}

状态转移方程

f(n) 是由 f(n-1) 和 f(n-2) 相加转移而来：

状态转移方程直接意味着暴力解法。
动态规划问题最困难的就是写出状态转移方程。

进一步优化，把时间复杂度将为O(1)

int fib(int n) {
    if (n == 2 || n == 1)
        return 1;
    int prev = 1, curr = 1;
    for (int i = 3; i <= n; i++) {
        int sum = prev + curr;
        prev = curr;
        curr = sum;
}
    return curr;
}
